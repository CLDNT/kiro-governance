Risk
• Sandbox AWS account contains shared data; broad access is 'chaotic' and may expose or contaminate project data — requested to move app to a non-sandbox account or request appropriate AWS account for Quik governance.
• Cross-account access complexity: app may need to be deployed or accessed across AWS accounts, requiring explicit account selection and possibly account requests; risk of deployment/access failures if not resolved.
• Meetings and artifacts outside Kiro (e.g., kickoff, review meetings, transcripts) are not automatically tracked; risk of missing evidence for governance gates and inaccurate project state.
• Oscar covers presales only and does not include delivery-phase transcripts; risk of missing delivery meeting artifacts unless Avoma or manual meeting links/transcripts are integrated.
• Unclear linking/identification of meetings to projects (no consistent keyword); risk of incorrect or missing associations unless meetings are manually linked via the app or a naming convention is enforced.

Application Architecture Design
• App to be built in QUIC that reads project data from existing Kiro database and supports cross-account access if needed.
• Interactive application (not just a reporting dashboard) to manage workflow, upload artifacts, and show project gates and artifacts per project.

Data Access and Security
• Avoid using Sandbox account for production app; use a controlled account (e.g., Crunch account) to prevent chaotic wide access.

Governance Points Tracking
• Capture meeting-based artifacts (kickoff, internal reviews) with links/transcripts as evidence tied to project gates.
• Track macro gates (e.g., SRS validated/approved, design docs approved, implementation plan approved, code approved, UAT report approved, run books approved, project docs approved).

Integration Requirements
• Integrate Avoma meeting links/transcripts for transcript analysis and criteria-based topic coverage checks.
• Leverage Oscar only for presales transcripts; project-phase transcripts reside in delivery and should be fetched separately.

Development Methodology Considerations
• App should reflect micro (Kiro-tracked tactical flags) vs. macro (project-level milestones) distinction; dashboard for reporting sits above interactive app.
